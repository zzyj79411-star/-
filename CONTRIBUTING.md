# 贡献指南

感谢你对 Lucid Brand Auditor 的关注！我们欢迎各种形式的贡献。

## 如何贡献

1. **提交 Issue**：如果你发现了 Bug 或有新功能建议，请提交 Issue。
2. **提交 Pull Request**：
    - Fork 本仓库。
    - 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)。
    - 提交你的更改 (`git commit -m 'Add some AmazingFeature'`)。
    - 推送到分支 (`git push origin feature/AmazingFeature`)。
    - 开启一个 Pull Request。

## 开发规范

- 使用 TypeScript 编写代码。
- 遵循现有的代码风格和组件结构。
- 确保新增功能有相应的类型定义。

## 行为准则

请保持专业和友好的交流氛围。
